# 日誌去識別化工具 v2 — 使用說明

把日誌裡的中文、IP（v4/v6）、網域、Email、跨 OS 使用者名、員工編號、
公司名換成假名後，才能安全地上傳給外部分析。產出可上傳的去識別檔，
以及一份留在本地的還原對照表。

---

## 第一次使用（安裝）

把 `setup_and_run.py` 和 `deidentify.py` 放在同一個資料夾，然後：

```
python setup_and_run.py
```

它會自動檢查並安裝所需套件（安裝最新版，只從官方 PyPI），安裝後做一次
載入驗證確認彼此相容。看到「環境就緒」即完成。需要 Python 3.9 以上。

---

## 日常使用

```
python setup_and_run.py 你的檔案.log
```

支援格式：`.txt` `.log` `.json` `.csv`（其他副檔名一律當純文字處理）。

選項：

**常用（每次都可能需要判斷）**

```
--keep-public-ip    保留公網 IP（分析外部攻擊來源時用）
                    → 預設會遮蔽公網 IP；若報告主題是「哪些外部 C2 IP 攻進來」
                      就加這個，保留 IP 才有分析價值。

-w 關鍵字           手動指定要遮蔽的詞（可多次使用）
                    → 適合自動偵測抓不到的名稱，例如沒有 Ltd/Corp 等後綴的公司名、
                      專案代號、機台名稱、人名等。
                    → 範例：-w "Acme" -w "PROJECT_X" -w "server01"
```

**偶爾用（特殊情境）**

```
--redact-uuid       啟用 UUID 遮蔽（預設關閉）
                    → UUID 通常是事件 ID，遮掉後難以在不同工具間交叉比對。
                      只有 UUID 本身就是識別個人/設備的主鍵時才開啟。

--no-dedup          關閉重複事件過濾（預設會去重）
                    → SentinelOne CSV 同一事件常重複出現，預設自動去重。
                      若你需要保留原始筆數（如做統計計算）才加這個。

--max-mb N          單檔輸出上限（MB），預設 10
                    → 超過上限會自動分割成 _part1、_part2…
                      檔案很小時不需要動；上傳有大小限制的平台才需要調整。
```

**很少用（進階路徑控制）**

```
--output-dir DIR    指定去識別化結果存放資料夾（預設：deidentified_output/）
--mapping-dir DIR   指定還原對照表存放資料夾（預設：deidentify_mapping/）
                    → 兩個通常一起用，用於多人共用環境或有特定資料夾規範時。

--yes               跳過安裝確認提示（自動化 / CI 用）
```

範例：`python setup_and_run.py *.csv --redact-uuid`

多關鍵字範例：`python setup_and_run.py incident.log -w "Acme Corp" -w "LAB-SERVER"`

## 產出的檔案

| 檔案 | 用途 | 可否上傳 |
|------|------|---------|
| `檔名_deidentified.<副檔名>` | 去識別後的結果 | ✅ 這份才可上傳 |
| `檔名_mapping.json` | 還原對照表 | ❌ 留本地，絕不上傳 |

執行後會印出稽核摘要，列出每類替換了幾個值。

---

## 偵測項目

| 項目 | 說明 |
|------|------|
| 中文 | 一律遮蔽 |
| IPv4 | 內外網分流；自動剔除瀏覽器版本號（如 Chrome/148.0.0.0 不會被誤判） |
| IPv6 | 含 fe80:: 連結本地位址 |
| 網域 | 真實 TLD 結尾；公開網域（google/microsoft 等）保留 |
| Email | 全遮 |
| 使用者名 | Windows `\Users\`、macOS `/Users/`、Linux `/home/` 三種路徑 |
| 員工編號 | 1 字母 + 6~8 數字（如 N1410360） |
| 公司名 | 後綴錨定（Ltd/Corp/Inc/Co.,Ltd/GmbH 等）自動偵測 |
| UUID | 預設不遮，`--redact-uuid` 開啟 |
| 密碼/Token | 遮蔽並警告（這類憑證應改密碼，不是只遮蔽） |

各格式處理：txt/log 逐行；json 只遮 value 保留 key；csv 保留標題列。

## 上傳前 5 秒檢查

```
□ 上傳的是 _deidentified 檔，不是原檔
□ 稽核摘要的替換數量看起來合理
□ 對照表 _mapping.json 留在本地，沒被一起上傳
□ 若出現密碼/Token 警告 → 該憑證請改密碼
```

## 限制（務必人工抽查）

- **公司名偵測靠後綴**：只能抓有 Ltd/Corp/Inc 等後綴的名稱；無後綴的公司名
  （如單一品牌名）抓不到。偶有誤殺（大寫開頭詞 + 後綴）。
- **純數字不抓**：時間戳、序號等純數字不遮，避免破壞分析價值。
- **IP 被遮後**：若要核對外部連線是否為惡意 C2，需用對照表還原 IP 自行比對。
- 任何自動工具都非 100%，上傳前請看稽核摘要並 grep 抽查已知敏感值。

## 想調整偵測規則

開 `deidentify.py`，最上方「偵測規則」區塊就是所有 Pattern：
- 公司後綴清單 → 改 `CORP_SUFFIX`
- 員工編號格式 → 改 `EMPID_PATTERN`
- 保留某公開網域 → 加進 `PUBLIC_DOMAIN_WHITELIST`
